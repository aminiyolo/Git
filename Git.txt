git init => 깃 사용 시작 //
(ls -al) : 보이지 않는 파일들 보이게 함 //
git add => working area에서 staging area로 보냄 / 
이것을 삭제하기 위해서는
git rm --cached <File> //

깃 상태를 알기 위해서는 git status //

Repository로 커밋 => git commit -m "commit message" // 
git log => 커밋의 기록 확인  //

git add와 commit 동시에 하기
=> git commit -am "commit message"
(한번이라도 commit을 한 대상에 대해서만 가능) //

로컬 저장소를 원격 저장소와 연결 하기 위해서는
git remote add origin <URL>
git branch -M main
git push -u origin main //

원격저장소에 푸쉬한 깃을 수정할 시 총 3가지 옵션이 있다. (git reset 사용)
- git reset --hard HEAD^ //  working directory, staging area, repository에 있는 것들 삭제
- git reset --mixed HEAD^ // staging area, repository에 있는 것들 삭제 ( default value === git reset HEAD^ ) 
- git reset --soft HEAD^ // 오로지 repository에 있는 것들 삭제
HEAD: 가장 최근 버전에서 , ^: 하나만 되돌림을 의미 // 

git branch 관리방법
- git branch 명령어 사용시 현재 가지고 있는 브랜치들을 확인할 수 있다.
- 기본적으로 main 브랜치인데(master -> main으로 바뀜) 새로운 브랜치를 추가하기 위해서는 명령어 
 (새로운 브랜치를 추가하기 위해서는 최소 1번 커밋을 해야한다)
=> git branch <Name>

-git branch 변경은 git checkout <Name>
깃을 병합할 때는 '병합의 결과'가 되는 대상에 checkout (ex. git checkout master => 마스터 브랜치에 병합하겟다.)
그 후 깃을 합치는 명령어는 git merge <합치려는 branchName> ( 마스터 브랜치에 병합이 된다.)

-git diff ( 변경 내역들끼리의 비교 결과를 보여줌 )
git log를 실행시키면 commit 옆에 나타나는 문자열은 commit hash //
git diff를 이용하여 커밋 간 비교하기 위해서는 
=> git diff <기준 commit> <비교대상 commit> //
실행 시키면 달라진 부분에 대해 앞에 +표시 됨  // ex) +<content> 

원격 저장소와 로컬저장소를 비교하기 위해서는 
=> git diff <기준 branch 이름> origin/<branch 이름>

이전 commit과 전전 commit을 비교하기 위해서는 
=> git diff HEAD HEAD^ 실행 ( HEAD는 현재 branch의 가장 최근 commit, HEAD^는 현재 branch의 가장 최근 commit에 하나 이전 commitd을 의미)
HEAD에서 ^의 개수는 현재 commit에서 몇번째 떨어진 commit인지를 의미 // ex) HEAD^^^ 최근 커밋에서 3개 이전 commit을 의미
다른점은 내용 앞에 +표시가 붙음

이전 commit과 현재 수정된 내용을 비교하기 위해서는
git diff HEAD 실행 //

브랜치간 비교를 하기 위해서는
git diff <기준 branch 이름> <비교 branch 이름>

git revert ( it is almost same as git reset )
=> git revert <돌아가고 싶은 commit> //
what's the difference ? => git reset과 git revert 모두 과거로 되돌려주지만, git reset은 되돌린 버전 이후의 버전들이 모두 사라짐
하지만, git revert는 되돌린 버전 이후의 버전들은 모두 유지되고 revert 되었다는 사실을 담은 commit이 새로 추가됨
